Core Principes

🔑 Geen hardcoded branding

Fontnamen, kleuren, posities — alles uit YAML configuratie. Nooit in Python code.

🏢 Tenant = bedrijf, Brand = huisstijl

Eén tenant kan meerdere brands hebben. Brand bepaalt visuele identiteit.

📐 Pixel-precise coördinaten

Alle posities geëxtraheerd uit referentie PDF's met PyMuPDF (pt → mm conversie).

📄 Stationery = achtergrond

Statische grafiek als PDF achtergrond. Engine rendert alleen variabele data.

Tenant Isolatie

🔒 Data Isolatie

Tenant wordt resolved uit template naam prefix. API leest alleen bestanden uit eigen tenant directory.

🖋 Font Isolatie

Fonts worden per-tenant geregistreerd bij ReportLab via font_files in brand.yaml.

📑 Stationery Isolatie

Elke tenant heeft eigen stationery PDF's. Engine zoekt alleen in tenants/{tenant}/stationery/.

Tenant Resolution Flow

// API ontvangt:
{ "template": "customer_bic_factuur" }

// _resolve_tenant_and_template():
1. Split op eerste underscore → tenant="customer", template="bic_factuur"
2. Check: tenants/customer/ exists? ✓
3. Check: tenants/customer/templates/bic_factuur.yaml exists? ✓
4. Laad brand.yaml → registreer fonts → init TemplateEngine
5. data_transform(json) → engine.build() → PDF

Stap 1: Data Transform

# Input: geneste JSON met sections/content blocks
{
  "sections": [{
    "content": [{
      "type": "bic_table",
      "sections": [{ "title": "BIC controles", "rows": [...] }]
    }]
  }]
}

# Output: flat dict met dot-notatie paden
{
  "bic":          { "aantal_conform": "6", "kosten_conform": "€ 1.860,00" },
  "reiniging":     { "aantal_conform": "3", "kosten_conform": "€ 525,00" },
  "samenvatting":  { "totaal_conform": "€ 3.375,00" },
  "detail_items":  [ { "BIC Controle nummer": "BIC-2025-..." } ],
  "client":        { "name": "Haagwonen" },
  "location":      { "name": "Strandbaak Kijkduin" }
}

Stap 2: YAML Bind Resolution

# YAML text_zone:
- bind: bic.kosten_conform      # → data["bic"]["kosten_conform"] → "€ 1.860,00"
  x_mm: 148.1
  y_mm: 50.5                      # top-down mm
  font: body                      # → brand.yaml fonts.body → "ArialMT"
  color: "text"                   # → brand.yaml colors.text → "#000000"

# Engine resolvet:
1. bind → data dict lookup
2. font "body" → brand.fonts["body"] → register_tenant_fonts()
3. color "text" → brand.colors["text"] → HexColor
4. y_mm → pt → rl_y = page_height - y_pt - font_ascent
5. canvas.drawRightString(x, rl_y, "€ 1.860,00")

Stap 3: Pagina Opbouw (4 lagen)

Layer 1: Stationery

Achtergrond PDF met logo's, kleurbalken, statische grafiek. Pixel-perfect als onderlaag.

Layer 2: Line Zones

Decoratieve lijnen (sectie-scheiding, subtotaal-lijnen). Kleur + dikte uit YAML.

Layer 3: Text Zones

Labels (static) + waarden (data-bound). Font, grootte, kleur, alignment uit YAML.

Layer 4: Tables

Kolom-gebaseerde tabellen met auto-paginering. Tekst truncation bij overflow (…).

Referentie PDF analyseren met PyMuPDF

Extract alle tekstelementen met posities, fonts, kleuren en lijnen:

import fitz

doc = fitz.open("referentie.pdf")
for page in doc:
    for block in page.get_text("dict")["blocks"]:
        for line in block.get("lines", []):
            for span in line["spans"]:
                print(
                    f"{span['text']:30}"
                    f" x0={span['bbox'][0]:7.1f}"
                    f" y0={span['bbox'][1]:7.1f}"
                    f" font={span['font']}"
                    f" size={span['size']}"
                    f" color=#{span['color']:06x}"
                )
Noteer per tekst: x0, y0 (pt, top-down), font naam, font size, hex kleur. Dit zijn je YAML coördinaten.
Coördinaten converteren: pt → mm
# Conversie:
mm = pt × 0.352778

# Voorbeelden:
 55.2 pt  →  19.5 mm
419.9 pt  → 148.1 mm
128.9 pt  →  45.5 mm

# PAS OP: voeg +1.0mm toe aan alle y_mm (behalve footer)
# Engine rendert op baseline, extractie meet bbox-top.
# Exacte offset verschilt per font — test en fine-tune!
Stationery PDF's maken
✓ WEL in stationery

Logo's, kleurbalken, achtergrondvlakken, decoratieve lijnen, vaste labels die NOOIT veranderen

✗ NIET in stationery

Variabele tekst, data-driven waarden, paginanummers, labels die per document kunnen verschillen

Formaat: Exact A4 (595.3×841.9pt portrait of 841.9×595.3pt landscape). Geen variabele data!

Page type YAML's schrijven (pixel-exact)

Eén YAML per pagina-layout. Elke text_zone is één tekstveld op de pagina:

name: factuur_overzicht
stationery: factuur_bg.pdf

line_zones:
  - x0_mm: 17.5
    y_mm:  45.3          # Y positie top-down
    x1_mm: 80.2
    width_pt: 3.0
    color: "primary"     # → brand.yaml

text_zones:
  # Statisch label
  - bind: _static.Totaal excl. BTW
    x_mm: 19.5
    y_mm: 83.8
    font: heading
    size: 10
    color: "secondary"

  # Dynamisch veld
  - bind: totaal.bedrag
    x_mm: 190.6
    y_mm: 83.8
    align: right
    font: heading
    size: 10
    color: "primary"

  # Automatisch paginanummer
  - bind: _page_number
    x_mm: 140.1
    y_mm: 273.5
    font: heading
    size: 12
    color: "white"
Tabel configuratie (voor data-tabellen)
table:
  data_bind: detail_items       # list in data dict
  origin:
    x_mm: 21.9
    y_mm: 42.7                  # Bovenkant headers
  row_height_mm: 5.6
  max_y_mm: 172                # Ondergrens → page-break
  show_header: true
  header_font: heading
  header_color: "secondary"
  body_font: body
  body_color: "text"
  columns:
    - { field: "Nummer", header: "Nummer", width_mm: 42, align: left }
    - { field: "Bedrag", header: "Bedrag", width_mm: 22, align: right }
Field names moeten EXACT matchen met keys in je data dict (inclusief spaties en hoofdletters).
Template YAML schrijven (documentstructuur)
name: acme_corp_factuur        # MOET beginnen met tenant naam!
tenant: acme_corp

pages:
  - type: special    page_type: cover        orientation: portrait
  - type: fixed      page_type: overzicht    orientation: portrait
  - type: fixed      page_type: details      orientation: landscape  repeat: auto
  - type: special    page_type: backcover    orientation: portrait
Template naamgeving: {tenant}_{documenttype} — de API splitst op eerste underscore om tenant te resolven.
data_transform.py uitbreiden (indien nodig)

Alleen nodig als je JSON structuur afwijkt van het bestaande formaat. Als JSON al flat is (direct dot-notatie paden), is geen transform nodig.

Test JSON + lokaal genereren

Maak een test JSON in schemas/ en genereer lokaal. Vergelijk visueel met referentie.

Fine-tuning: visuele validatie
Positionering

Tekst op juiste Y? Vaak +1.0mm offset nodig (baseline vs bbox-top).

Fonts & Kleuren

heading vs body? secondary voor labels, text voor waarden?

Lijnen

0.75pt dun, 3.0pt dik? Start/eind x correct?

Tabellen

Kolommen breed genoeg? Tekst truncated met "…"?
